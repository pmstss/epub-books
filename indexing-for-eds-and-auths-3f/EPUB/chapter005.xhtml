Chapter 5

Index Length and Index Space

This chapter discusses two of the most important questions when dealing with indexes: knowing how long the index should be and reducing the size of an index (either by employing typographic means or actually reducing the length of the index) to fit the space available.

The first section in this chapter covers the average or appropriate length of an index given the type of material it covers. The remainder of the chapter deals with the thorny question of reducing the size of an index. First, techniques to “squeeze” an existing index are discussed. Next, methods for appropriately cutting an index are covered.

While there are a number of techniques that a professional indexer can use to reduce the size of an index, such cutting almost invariably reduces the usability of the index and makes specific information more difficult for the reader to find. So it behooves the editor to consider adding a signature to accommodate the index as it stands. Often tightening up the preliminary matter can save a page or two, which can then be allocated to the index.

AVERAGE/APPROPRIATE INDEX LENGTHS

In Indexing Books (Chicago: University of Chicago Press, 1993), Nancy Mulvany provides the following general guidelines for the length of an index:

Type of Material

Length of Index
Relative to Text
Mass market trade books

2–5%
General reference books

7–8%
Technical documentation

10%
Specialized technical material

15+%

The percentages represent the number of pages in the index relative to the number of pages in the text. Accordingly, a 250-page trade book should have an index in the range of 5–12 pages, while a reference book of the same length (250 pages) should have an index of 17–20 pages. The rising percentages in the chart reflect the increasing density of the material, requiring increasingly longer indexes.

Legal texts generally use a 5 percent figure for the size of their index. Scholarly material, because of its often dense arguments, benefits from an index that, like general references as noted in the previous table, is in the 7–8 percent range.

The length of the index is directly related to the number of entries the indexer is told to pick up:

  • Light indexing: 3 or fewer entries per page
  • Medium indexing (which most books get): 4–6 entries per page
  • Heavy indexing: 7–10 entries per page
  • Encyclopedic treatment: 10+ entries per page

The exhaustivity or depth of indexing should be appropriate for the material being indexed. Asking the indexer to work at 3 entries per page in dense scholarly material will do a disservice to both author and reader, as important topics will be left out of the index, making information finding inefficient, if not impossible.

REDUCING THE INDEX SPACE

One of the difficult tasks that an editor faces is having to reduce the size of a completed index to fit the space available. If the index is too long for the space, the first thing to consider is how much the size of the index needs to change. Reducing the index by only a few lines is significantly easier than cutting it to save one or more pages, and there are a number of space-saving techniques that can easily reduce the size of an index without cutting the index itself.

Tactics for Minimal Space Reductions

If only a few lines need to be saved, then it may be possible to tighten up the index using one or more of these techniques:

  • Modifying the letter spacing slightly to reduce the number of turnover lines
  • Closing up the space between individual alphabetical sections of the index
  • Eliminating section heading letters, if those have been used

There are a number of additional techniques that may be used to further reduce the space that an index takes without changing the length of the index itself.

Number of Columns

Changing the number of columns used in the index can significantly impact the number of pages required. Changing from a single- to double-column display can save 25 percent or more space in a large index. The reduction in index size is based on the fact that index entries often contain short, single-line entries. Accordingly, two columns make it possible to accommodate two entries on a line as opposed to having a single entry on that line in a one-column format.

A simple example illustrates this change:

Sai Jinhua incident, 440

Sai Jinhua (Xia Yan), 440

Saitô Mokichi, 102, 103

Sakaguchi Ango, 190, 191–192

“The Salamander” (Sanshôuo, Ibuse Masuji), 205

salarymen, 737

Typography Changes

Most books typeset indexes two point sizes smaller than the text; it is rare for an index to be in the same size as the text. If necessary to accommodate an index longer than the space available, the index font size may be reduced even further.

Certainly, consideration must be given to the age of the intended audience for the text when considering reducing font size. A book for seniors, for example, should rarely use a font size smaller than 12 point, even for the index. In general, a 1-point font-size reduction will result in an index shorter by about 10–15 percent.

It is also possible to reduce the amount of space an index takes by tightening letter spacing, which may reduce the number of turnover lines, or by reducing leading (line spacing). Both of these solutions, however, reduce readability, making the index harder to use and less inviting.

Indented vs. Paragraph Format

Chapter 4 thoroughly discusses the differences between indented and paragraph format. While indented index format is preferred for ease of readability, changing to a run-in format can reduce the index size by around 15 percent. Care should be taken when using run-in style, since it is difficult to use sub-subheadings and present them in a way that readers can understand.

Page Number Contractions

Chapter 4 also covers ways to contract page ranges. If those contractions are applied to the entire index, you may save some space by reducing the number of turnover lines. You may also decide to use page conflation more liberally, changing “11, 12, 13, 15” to “11–15.” Although this may affect usability somewhat, it is as easy to keep reading as it is to flip back and forth from the index to the text over and over again.

REDUCING INDEX LENGTH

It may be the case, however, that none of the methods discussed so far have served to sufficiently reduce the size of the index. In this case, the editor should ask the indexer to cut the index by a specified amount. Having the indexer do this will ensure that the overall structure of the index and the inter-entry relationships are maintained to the extent possible. An editor or author making changes willy-nilly can often cause cross-reference errors that indexers would be able to find and fix using their indexing software.

The process of cutting an index is not a simple matter. Reducing the size of the index by one page can mean having to cut 100 entries; that can represent a significant proportion of the index. The cutting must be done with care and with a well defined plan in mind, and it takes a lot of work to do properly. The following techniques can be used when cutting the index is necessary.

Number of Subheadings

Generally, indexers use subheadings to differentiate strings of more than five locators. However, the number of undifferentiated locators may be increased to six or seven to control the size of the index. Keep in mind that this change makes it more difficult for the reader to find specific information, as he must now search all seven references for a given topic rather than finding descriptive subheadings.

Depending on the number of headings this change affects, it can save considerable space when an entry contracts from:

censorship (Japan)

during Allied occupation, 184–185, 187, 222

of Hiratsuka Raichô, 96

laws on, 55

in 1930s, 56

of Tanizaki Jun’ichirô, 119

to

censorship (Japan), 55, 56, 96, 119, 184–185, 187, 222

Again, consideration must be given to the trade-off with index usability.

What else might you cut? Names with only one page number or concepts that seem tangential to the main argument of the book may be left out. For entries with multiple subheadings, check to be sure that there is no over-analysis; it may be possible to combine several subheadings into a slightly more general one.

Under a person’s name, you might have the following entries:

relationship with father, 27

relationship with sister, 15

relationship with wife, 92, 124

Those could easily be combined into one subheading:

family relationships, 15, 27, 92, 124

Another example of over-analysis is a heading with multiple subheadings that all reference the same small page range:

Arkansas

alcohol sales online in, 701, 702

sales tax on Internet purchases in, 699

tobacco sales online in, 700

That could easily be reduced to:

Arkansas, 699–702

Such ranges covering less than six pages are not a major hindrance to the reader and can help keep the index uncluttered. Subheadings, after all, do take extra time to read and if they all refer to the same small page range, the reader has wasted time for little additional information.

Finally, remove single subheadings, if any. Change:

lace cookies

baking techniques, 11

to

lace cookies, 11

Alternately, combine the heading and subheading into a “modified” heading (for example, “lace cookies, baking techniques, 11”), although the additional line length may mean a turnover line, so you haven’t actually saved any space.

Cross-References

One of the significant benefits of a good index is that it provides alternate terms by which users can find a specific subject:

Cancer. See Oncology

If necessary, some or even all of these see references can be removed. Depending on the size and complexity of the index, this modification might save a page or two. However, the findability of specific topics is significantly reduced, since the user must now try to think of possible alternate terms herself if the information is not found under the first term she thinks of.

Indexes also direct the reader to additional material that may possibly be relevant through the use of see also references. These, too, may be pruned to save space, although this action diminishes the index’s usefulness.

Double-Posts

Indexers will place identical locators under two or more different headings, allowing users to find the information in whichever of those places they look:

baby chairs, 27, 38, 197

…

highchairs, 27, 38, 197

…

infant chairs, 27, 38, 197

The duplicate entries may be removed to reduce the overall size of the index. However, think carefully about which term most of the book’s audience is likely to look for.

INCREASING INDEX LENGTH

In rare cases it may be necessary to increase the length of an index. Any of the typographic techniques mentioned earlier may be used. However, it is still necessary to keep in mind that a font size too large or lines spaced too far apart impair readability. So the changes should be moderate, if possible.

In addition, the techniques for reducing an index length can be used in reverse: increasing the number of double-posts, adding (appropriate) cross-references, and reducing the number of undifferentiated locators. However, these require additional work by the indexer, so they should not be undertaken lightly, as they will require an additional (usually hourly) fee for the adjustments to be made.
